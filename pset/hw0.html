A mini tutorial for the emacs editor

To write files from a Unix shell, we use a program called an editor. You may use your favorite one. If you are not familiar with any editor, I recommend emacs or vim.

To create a file with emacs, type the command emacs hello_world.txt. Type the phrase Hello world. You should see the words entered into the file. To save the file type Ctrl-x Ctrl-s. In emacs commands, "Ctrl-" is typically abbreviated as "C-" as in C-x C-s.

To be able to use emacs efficiently, you should familiarize yourself with a number of commands. For example:

  • Saving, quitting, and opening files:

    • C-x C-s save
    • C-x c quit
    • C-x C-w save file as
    • C-x C-f open file

  • Moving the cursor:

    • C-[right arrow] jump to next word
    • C-[left arrow] jump to previous word
    • C-[down arrow] jump next paragraph
    • C-[up arrow] jump previous paragraph

  • Copy and paste:

    • C-k kill (that is, cut) text between cursor and end of line
    • C-[spacebar] set copy region marker
    • C-w kill text between copy region marker and cursor
    • C-y yank (that is, paste) previously killed text
    • C-_ undo

  • Search:

    • C-s search forward
    • C-r search backward

See this tutorial for a more complete list of commands.

A mini tutorial for the vim editor

There are two fundamental modes in vim:

  • Command mode: everything you type is a command that allows you to move the cursor around, delete and copy text, search for words, save the file, quit, etc.
  • Insert mode: everything you type (save for special keys) is added to the file, as would happen in a word processor; however, you can only move the cursor with the arrow keys, not the mouse.

To create a file with vim, type the command vim hello_world.txt. The editor starts in command mode. We can switch to insert mode by typing i. You should see the message -- INSERT -- appear at the bottom of your window. To go back to command mode, press the escape key. The message should now disappear. Toggle back and forth between the two modes.

In insert mode, type the phrase Hello world. You should see the words entered into the file. To save the file, go to command mode, type :w and enter.

To be able to use vim efficiently, you must get used to the wealth of commands available in command mode. Some examples:

  • Saving and quitting:

    • :w save
    • :wq save and quit
    • :q quit
    • :q! quit, ignoring changes
    • :u undo

  • Moving the cursor:

    • 0 first column of the line
    • ^ first non-blank character of the line
    • $ jump to the last character of the line
    • w jump to next word
    • b jump to word-beginning
    • } jump over a block of lines
    • % jump to matching bracket
    • * go to next occurrence of word under cursor
    • # go to previous occurrence of word under cursor

  • Copy and paste:

    • yy copy or yank current line
    • 88yy copy 88 lines
    • p paste

  • Search:

    • /foo search the document for the next occurrence of foo
    • ?foo search the document for the previous occurrence of foo

See this tutorial for a more complete list of commands and a discussion of the visual mode (yet another useful mode!).

Git and GitHub (optional)

Git is a revision control system that has become dominant in software development. Revision control allows you to back up a set of files at different points in time. This is tremendously useful for software projects in which rapid iteration is necessary -- it allows you to try many things out and, if anything breaks, easily revert to a previous version.

Git can make your life a lot easier in the Kaggle competition, as you will be modifying and testing your prediction methods frequently as a team.

Git is already installed on rice. To use it in your computer, you will have to install it. You may find installers for Windows and Mac OSX here.

I recommend the following two resources to learn Git:

  • Git Tutorial: long, but very thorough.
  • Try Git: interactive tutorial to get started with Git in 15 minutes.

GitHub is a web hosting service for software development projects that use Git for revision control, making it easy for multiple people to collaborate on software projects in an organized fashion. It is also something of a social network for programmers.

To use GitHub, you will need to create an account on http://www.github.com and set up ssh keys on your computer. Instructions to do this can be found in this article.

In the Kaggle competition, you have several options. You may:

  1. Use a different revision control system, or no revision control system at all.
  2. Use Git on your computer or in rice.stanford.edu to develop your predictive methods individually.
  3. Host a project on Github that is administered by all the members of your team. Develop different branches of the code and merge them as necessary. For example, someone may write a script to transform or augment a group of variables, while another person tests different regression methods. You can seamlessly combine these two contributions into a single pipeline using Git.
